# GuatequeStore

## Plataforma de gestión y venta de productos de moda

El objetivo principal de este proyecto es desarrollar una plataforma completa para la gestión y venta de productos de moda. Esta plataforma simula el funcionamiento de una tienda de ropa real, permitiendo a los usuarios explorar el catálogo, consultar tallas y colores disponibles, gestionar un carrito de compras y completar pedidos.

Además, proporciona herramientas internas para que los administradores mantengan actualizado el inventario, gestionen productos y optimicen la experiencia de compra.

---

## Moda responsable y economía circular

### Transformando la industria de la moda

Guateque impulsa un modelo más responsable y sostenible, promoviendo la economía circular y reduciendo el desperdicio. Todos los productos provienen de proveedores que cumplen estándares ESG y fomentamos devoluciones responsables: las prendas devueltas, con recibo, pueden ser recicladas mientras los clientes obtienen recompensas, cerrando el ciclo de vida del producto.

## Estructura del proyecto

El repositorio está organizado de la siguiente manera:

- **Android/** → Aplicación móvil desarrollada en Java  
- **Frontend/** → Aplicación web desarrollada en React  
- **Backend/** → API y lógica del servidor desarrollados en Java con Spring Boot  

### Frontend

- **Framework:** React  
- Permite interactuar con el sistema, visualizar productos, proveedores y eventos.  
- Se puede incluir una captura de pantalla del frontend para mostrar la interfaz visual.

### Backend

- **Lenguaje y framework:** Java con Spring Boot  
- Expone APIs REST para gestionar productos, proveedores, pedidos y eventos.

### Android

- Aplicación móvil desarrollada en Java  
- Permite a los usuarios acceder a funcionalidades similares a la web desde dispositivos Android.

## Tecnologías utilizadas

- **Frontend:** React, JavaScript, HTML, CSS

## Cómo ejecutar el proyecto

- **Frontend:** instalar dependencias y ejecutar la app en un servidor local.  
- **Backend:** correr la aplicación Spring Boot para exponer las APIs.  
- **Android:** abrir el proyecto en Android Studio y ejecutar en un emulador o dispositivo físico.

---

## Buenas prácticas al integrar módulos

- Trabaja en ramas separadas por módulo y luego integra cada módulo a la carpeta correspondiente (`Android/`, `Frontend/`, `Backend/`).  
- Antes de subir cambios, revisar siempre `git status` y diferencias para evitar sobrescribir trabajo remoto.
